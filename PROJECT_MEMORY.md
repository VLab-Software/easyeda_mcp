# Project Memory: EasyEDA Pro MCP Bridge

## Goal

Build a live MCP integration for EasyEDA Pro so Codex can inspect and assist the currently open EasyEDA Pro project through an installed EasyEDA extension.

The integration is extension-only. There is no offline `.epro` parser in the current direction.

## Current Architecture

- MCP server: TypeScript/Node, stdio transport.
- Local bridge: WebSocket server at `ws://127.0.0.1:8765`.
- EasyEDA Pro extension: connects to the local bridge using EasyEDA Pro `SYS_WebSocket`.
- The MCP calls extension handlers, and the extension calls EasyEDA Pro `eda.*` APIs.

Important files:

- `src/index.ts`: MCP server entrypoint.
- `src/bridge/EasyEdaBridge.ts`: WebSocket bridge.
- `src/mcp/registerTools.ts`: MCP tool registration.
- `extension/src/index.ts`: EasyEDA Pro extension handlers.
- `src/schematic/analysis.ts`: generic schematic normalization and analysis.
- `scripts/package-extension.mjs`: packages `.eext` using JSZip.

## Extension Packaging Notes

EasyEDA Pro expects a `.eext` package, not a raw folder.

The working package is generated by:

```powershell
npm run build:extension
npm run package:extension
```

Current extension package:

```text
C:\Users\vlabsoft\Documents\easyeda_mcp\build\dist\easyeda-mcp-bridge_v0.1.2.eext
```

Important packaging fixes already made:

- `extension.json` must be at package root.
- `name` must use lowercase/hyphen style, not underscores.
- `uuid` must be exactly 32 lowercase alphanumeric characters.
- Extension bundle must be built like the official EasyEDA SDK:
  - `esbuild`
  - `format=iife`
  - `globalName=edaEsbuildExportName`
- EasyEDA disables external interaction permission by default when the extension uses WebSocket/file APIs. User must enable that permission.
